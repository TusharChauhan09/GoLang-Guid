Introduction & Setup

Go (often called Golang) is a statically typed, compiled, garbage-collected programming language designed at Google by Robert Griesemer, Rob Pike, and Ken Thompson. It emphasizes simplicity, fast compilation, built-in concurrency, and a strong standard library.

Key Characteristics

  • Statically typed — types are checked at compile time.
  • Compiled — produces a single static binary, no runtime VM needed.
  • Garbage collected — automatic memory management.
  • Concurrent — goroutines and channels are first-class.
  • Simple — only ~25 keywords, no inheritance, no exceptions.
  • Cross-platform — easy cross-compilation with GOOS/GOARCH.

Installation

Download Go from go.dev/dl. After install, verify:

# Check version
$ go version
go version go1.22.0 linux/amd64

# Check environment
$ go env GOPATH GOROOT

Your First Program

Create a file hello.go:

package main

import "fmt"

func main() {
    fmt.Println("Hello, World!")
}

Run it:

$ go run hello.go
Hello, World!

# Or compile to a binary
$ go build hello.go
$ ./hello
Every Go file must start with a package declaration. The entry point of an executable is func main() inside package main.

Program Structure

Every Go program follows a fixed skeleton: a package declaration, imports, then top-level declarations (constants, variables, types, functions).

Anatomy of a Go File

// 1. Package clause — required, first non-comment line
package main

// 2. Import declarations
import (
    "fmt"
    "math"
    "strings"
)

// 3. Top-level declarations (any order)
const Pi = 3.14159

var Greeting = "Hello"

type Point struct { X, Y int }

func add(a, b int) int { return a + b }

func main() {
    fmt.Println(Greeting, strings.ToUpper("go"))
    fmt.Println(math.Sqrt(16))
}

Identifiers & Naming Rules

  • Must start with a letter or underscore, followed by letters/digits/underscores.
  • Case-sensitive: Foo ≠ foo.
  • Capitalization controls visibility: identifiers starting with an uppercase letter are exported (public), lowercase are unexported (package-private).
  • Idiomatic names: short, camelCase for locals, PascalCase for exported.

The 25 Keywords of Go

break       default      func         interface    select
case        defer        go           map          struct
chan        else         goto         package      switch
const       fallthrough  if           range        type
continue    for          import       return       var

Comments

// Single-line comment

/*
   Multi-line comment.
   Used for docs at the top of packages.
*/

// Doc comment for an exported identifier (becomes godoc).
func Add(a, b int) int { return a + b }

Semicolons

Go uses semicolons internally, but they are inserted automatically by the lexer at line breaks. You almost never write them. The consequence: an opening { must appear on the same line as the keyword that starts the block.

// CORRECT
if x > 0 {
    fmt.Println(x)
}

// COMPILE ERROR — brace on next line
if x > 0
{
    fmt.Println(x)
}

Variables & Constants

Go offers several ways to declare variables. Type inference makes most declarations short, but the long forms exist for package-level declarations and explicit typing.

Variable Declaration — Four Forms

// 1. Full form: var name type = value
var age int = 25

// 2. Type inferred from value
var name = "Alice"

// 3. Zero value (no initializer)
var count int          // 0
var active bool        // false
var title string       // ""

// 4. Short declaration (only inside functions!)
score := 100
city, country := "Tokyo", "Japan"
The short form := can only be used inside functions. At package level you must use var.

Multiple Declarations

// Group with parentheses
var (
    name    string = "Bob"
    age     int    = 30
    height  float64 = 1.78
)

// Multiple on one line
var x, y, z int = 1, 2, 3

// Parallel assignment / swap
a, b := 1, 2
a, b = b, a   // swap, no temp variable needed

Zero Values

Every variable, if not initialized, gets the zero value of its type — never garbage memory.

TypeZero Value
numeric (int, float64, …)0
boolfalse
string""
pointer, slice, map, chan, func, interfacenil
structeach field's zero value

The Blank Identifier _

Use _ to discard a value you must receive but don't need.

_, err := fmt.Println("hi")   // ignore the byte count
for _, v := range nums { ... }   // ignore the index

Constants

Constants are immutable values known at compile time. They have no address and cannot be assigned to.

const Pi = 3.14159
const Greeting string = "Hello"

const (
    StatusOK       = 200
    StatusNotFound = 404
)

Typed vs Untyped Constants

const a = 10          // untyped — flexible, becomes whatever type fits
const b int = 10      // typed — must be assigned to int

var f float64 = a    // OK, a is untyped
var g float64 = b    // COMPILE ERROR

iota — The Constant Generator

iota is a special identifier that increments by one inside a const (...) block, starting at 0.

const (
    Sunday = iota   // 0
    Monday              // 1
    Tuesday             // 2
    Wednesday           // 3
)

// Bit flags using iota
const (
    Read    = 1 << iota  // 1
    Write                  // 2
    Execute                // 4
)

Output Functions

Control Flow

Go has only three control structures: if, for, and switch. There is no while or do-while — for handles all loops.

if / else if / else

if x > 0 {
    fmt.Println("positive")
} else if x < 0 {
    fmt.Println("negative")
} else {
    fmt.Println("zero")
}

// if with init statement — variable scoped to the if
if v, err := parse(s); err == nil {
    fmt.Println(v)
} else {
    fmt.Println("bad input:", err)
}
Conditions are not wrapped in parentheses, but braces are always required — even for single-line bodies.

for — The Only Loop

// 1. Classic three-clause
for i := 0; i < 10; i++ {
    fmt.Println(i)
}

// 2. Condition-only (acts like while)
n := 10
for n > 0 {
    n--
}

// 3. Infinite loop
for {
    if done() { break }
}

// 4. for-range over collections
nums := []int{10, 20, 30}
for i, v := range nums {
    fmt.Println(i, v)
}

// for-range over a string yields rune index + rune
for i, r := range "héllo" { ... }

// for-range over a map (unordered)
for k, v := range myMap { ... }

// for-range over a channel
for v := range ch { ... }

// Go 1.22+: range over an integer count
for i := range 5 {           // i = 0..4
    fmt.Println(i)
}

break, continue, labels, goto

Outer:
for i := 0; i < 5; i++ {
    for j := 0; j < 5; j++ {
        if j == 3 { break Outer }      // exits both loops
        if j == 1 { continue }          // next inner iteration
    }
}

// goto — exists, rarely needed, jumps to a label in the same function
goto End
End:

switch

// Cases do NOT fall through by default
switch day {
case "Sat", "Sun":                // multiple values
    fmt.Println("weekend")
case "Mon":
    fmt.Println("monday blues")
    fallthrough                  // explicit fall-through
case "Tue":
    fmt.Println("midweek")
default:
    fmt.Println("other")
}

// switch with no expression — like if/else if chains
switch {
case score >= 90: grade = "A"
case score >= 80: grade = "B"
default:          grade = "F"
}

// switch with init
switch v := getValue(); v {
case 1: ...
}

// type switch (covered more in §16)
switch v := i.(type) {
case int:    fmt.Println("int", v)
case string: fmt.Println("string", v)
default:      fmt.Println("other")
}

Functions

Functions are first-class values: they can be assigned to variables, passed as arguments, and returned from other functions.

Basic Syntax

func name(param1 type1, param2 type2) returnType {
    return value
}

func add(a int, b int) int {
    return a + b
}

// Same-type params can share their type
func add(a, b int) int { return a + b }

Multiple Return Values

func divmod(a, b int) (int, int) {
    return a / b, a % b
}

q, r := divmod(17, 5)

Named Return Values

func split(sum int) (x, y int) {
    x = sum * 4 / 9
    y = sum - x
    return     // "naked" return — uses x and y
}
Named returns are great for documentation and for use with defer to modify return values, but in long functions they hurt readability.

Variadic Functions

func sum(nums ...int) int {
    total := 0
    for _, n := range nums { total += n }
    return total
}

sum(1, 2, 3)
nums := []int{1, 2, 3}
sum(nums...)              // spread a slice

Functions as Values

var op func(int, int) int = add
fmt.Println(op(3, 4))   // 7

// Anonymous function
greet := func(name string) { fmt.Println("hi", name) }
greet("Ana")

// IIFE — immediately invoked
func() { fmt.Println("once") }()

Closures

func counter() func() int {
    n := 0
    return func() int {
        n++
        return n
    }
}

c := counter()
fmt.Println(c(), c(), c())   // 1 2 3

Higher-Order Functions

func apply(nums []int, fn func(int) int) []int {
    out := make([]int, len(nums))
    for i, n := range nums { out[i] = fn(n) }
    return out
}

doubled := apply([]int{1, 2, 3}, func(x int) int { return x * 2 })

init() — Special Function

Each file may define an init() that runs once, before main(), to initialize package state.

func init() {
    fmt.Println("package initialized")
}

Arrays

An array has a fixed length that's part of its type. Arrays are passed by value (copied). In practice, you almost always use slices instead — but arrays are the foundation slices are built on.

Declaration

var a [5]int                          // [0 0 0 0 0]
var b = [3]string{"a", "b", "c"}
c := [...]int{10, 20, 30}            // length inferred = 3
d := [5]int{1: 10, 3: 30}            // [0 10 0 30 0]

Access & Length

a[0] = 100
fmt.Println(a[0], len(a))

Arrays Are Values

x := [3]int{1, 2, 3}
y := x          // FULL COPY
y[0] = 99
fmt.Println(x, y)   // [1 2 3] [99 2 3]

Multi-dimensional

var grid [3][3]int
grid[1][2] = 5

matrix := [2][2]int{
    {1, 2},
    {3, 4},
}

Slices

A slice is a dynamically-sized, flexible view into the elements of an array. It's the workhorse data structure of Go. A slice has three parts: pointer (to a backing array), length, and capacity.

Creating Slices

// 1. Slice literal
s := []int{1, 2, 3}

// 2. From an array — s = a[low:high]  (high is exclusive)
a := [5]int{10, 20, 30, 40, 50}
s2 := a[1:4]      // [20 30 40], len=3, cap=4

// 3. make(type, length, capacity)
s3 := make([]int, 5)        // len=5, cap=5, zeroed
s4 := make([]int, 5, 10)    // len=5, cap=10

// 4. nil slice
var s5 []int             // nil; len=0, cap=0

len() vs cap()

s := make([]int, 3, 5)
fmt.Println(len(s), cap(s))   // 3 5

Slicing a Slice

s := []int{0, 1, 2, 3, 4}
s[1:4]          // [1 2 3]
s[:2]           // [0 1]   — from start
s[3:]           // [3 4]   — to end
s[:]            // full slice

// 3-index slice — limits capacity (security/safety)
s[1:3:3]        // len=2, cap=2

append

s := []int{1, 2}
s = append(s, 3)              // [1 2 3]
s = append(s, 4, 5, 6)        // [1 2 3 4 5 6]

// Append a slice to a slice
more := []int{7, 8}
s = append(s, more...)
When append exceeds capacity, Go allocates a new larger backing array. Always reassign: s = append(s, x).

copy

src := []int{1, 2, 3}
dst := make([]int, 3)
n := copy(dst, src)   // returns number of elements copied

Slices Share Backing Arrays

a := []int{1, 2, 3, 4}
b := a[1:3]
b[0] = 99
fmt.Println(a)   // [1 99 3 4]  ← a was modified!

Common Idioms

// Remove element at index i (preserves order)
s = append(s[:i], s[i+1:]...)

// Insert v at index i
s = append(s[:i], append([]int{v}, s[i:]...)...)

// Stack: push / pop
stack = append(stack, x)         // push
x, stack = stack[len(stack)-1], stack[:len(stack)-1]   // pop

// Reverse
for i, j := 0, len(s)-1; i < j; i, j = i+1, j-1 {
    s[i], s[j] = s[j], s[i]
}

Maps

A map is an unordered collection of key/value pairs. Internally a hash table. Keys must be of a comparable type (anything except slices, maps, and functions).

Declaration & Initialization

// 1. make
m := make(map[string]int)

// 2. Map literal
ages := map[string]int{
    "Alice": 30,
    "Bob":   25,
}

// 3. nil map — read-only, write panics
var m2 map[string]int

Operations

ages["Carol"] = 28          // add or update
age := ages["Alice"]         // read
delete(ages, "Bob")         // remove
n := len(ages)               // size

The "comma ok" Idiom

v, ok := ages["Dan"]
if ok {
    fmt.Println("Dan is", v)
} else {
    fmt.Println("Dan not found")
}
Reading a missing key returns the zero value of the value type — no error. Use the , ok form to distinguish "missing" from "zero".

Iteration

for k, v := range ages {
    fmt.Printf("%s: %d\n", k, v)
}
Map iteration order is randomized by the runtime. Don't depend on it. To iterate in a specific order, sort the keys first.

Sorted Iteration

import "sort"

keys := make([]string, 0, len(ages))
for k := range ages { keys = append(keys, k) }
sort.Strings(keys)
for _, k := range keys {
    fmt.Println(k, ages[k])
}

Sets via Maps

set := map[string]struct{}{}
set["apple"] = struct{}{}
set["banana"] = struct{}{}
_, has := set["apple"]    // true

Strings & Runes

A Go string is an immutable read-only slice of bytes — typically UTF-8 encoded. byte is an alias for uint8 (raw byte). rune is an alias for int32 (a Unicode code point).

Basics

s := "hello"
len(s)               // 5 (bytes)
s[0]                // 104 (byte 'h')

// strings are immutable
s[0] = 'H'           // COMPILE ERROR

// concatenation
greeting := "hi " + "there"

Raw vs Interpreted Literals

a := "line1\nline2"      // interpreted: \n is newline
b := `line1\nline2`       // raw: \n is two chars

Escape Sequences

\nnewline\ttab
\rcarriage return\\backslash
\"double quote\'single quote (rune lits)
\xFFhex byte\u00E94-digit unicode

Iterating Correctly Over a String

s := "héllo"

// BYTE iteration — wrong for non-ASCII
for i := 0; i < len(s); i++ {
    fmt.Printf("%d: %x\n", i, s[i])
}

// RUNE iteration via for-range — correct
for i, r := range s {
    fmt.Printf("%d: %c (%U)\n", i, r, r)
}

Conversions

s  := "hello"
bs := []byte(s)            // to byte slice
rs := []rune(s)            // to rune slice
back := string(rs)          // back to string
string(65)                   // "A" (rune 65)

The strings Package

import "strings"

strings.Contains("hello", "ell")        // true
strings.HasPrefix("hello", "he")        // true
strings.HasSuffix("hello", "lo")        // true
strings.Index("hello", "l")             // 2
strings.Count("banana", "a")            // 3
strings.Replace("foo", "o", "0", -1)    // "f00"
strings.Split("a,b,c", ",")           // ["a","b","c"]
strings.Join([]string{"a","b"}, "-")      // "a-b"
strings.ToUpper("go")                  // "GO"
strings.ToLower("Go")                  // "go"
strings.TrimSpace("  hi  ")            // "hi"
strings.Repeat("ab", 3)                 // "ababab"

Building Strings Efficiently

var b strings.Builder
b.WriteString("hello ")
b.WriteString("world")
result := b.String()

strconv — String / Number

import "strconv"

n, err := strconv.Atoi("42")         // string -> int
s := strconv.Itoa(42)               // int -> string
f, _ := strconv.ParseFloat("3.14", 64)
b, _ := strconv.ParseBool("true")

Pointers

A pointer holds the memory address of a value. Go has pointers but no pointer arithmetic — they're for sharing, not for raw memory tricks.

Address & Dereference

x := 10
p := &x              // p is *int, holds address of x
fmt.Println(*p)      // 10  (dereference)
*p = 20              // modifies x through p
fmt.Println(x)       // 20

Declaration

var p *int           // nil — declared but not pointing
var q = new(int)      // allocates a zeroed int, returns *int
*q = 5

Pointers as Function Arguments

func double(n *int) {
    *n = *n * 2
}

x := 3
double(&x)
fmt.Println(x)   // 6

Why Pointers?

  • To mutate a value passed to a function.
  • To avoid copying large structs.
  • To represent optional values (use nil as "absent").

What Go Doesn't Allow

p := &x
p++                 // COMPILE ERROR — no pointer arithmetic
p = p + 1           // COMPILE ERROR

Pointers to Structs (Auto-deref)

type Point struct { X, Y int }
p := &Point{X: 1, Y: 2}
fmt.Println(p.X)         // 1 — Go auto-dereferences
fmt.Println((*p).X)      // same thing, explicit

Structs

A struct is a typed collection of named fields — Go's composite type for grouping data. Combined with methods, structs replace classes from other languages.

Defining & Instantiating

type Person struct {
    Name string
    Age  int
}

// Different ways to create
var p1 Person                          // zero values: "" 0
p2 := Person{"Alice", 30}              // positional — fragile
p3 := Person{Name: "Bob", Age: 25}     // named — preferred
p4 := Person{Name: "Carol"}             // Age = 0 (zero value)
p5 := &Person{Name: "Dan"}              // pointer to struct

Field Access

p3.Name = "Bobby"
fmt.Println(p3.Age)

Anonymous Structs

config := struct {
    Host string
    Port int
}{
    Host: "localhost",
    Port: 8080,
}

Embedded Fields (Composition)

Go has no inheritance. Instead, you embed a type — its fields and methods are promoted.

type Animal struct {
    Name string
}

func (a Animal) Speak() { fmt.Println(a.Name, "makes a sound") }

type Dog struct {
    Animal      // embedded
    Breed string
}

d := Dog{Animal: Animal{Name: "Rex"}, Breed: "Lab"}
fmt.Println(d.Name)   // promoted from Animal
d.Speak()              // promoted method

Struct Tags

Backtick strings after a field — read by reflection (e.g. by JSON encoders).

type User struct {
    Name  string `json:"name"`
    Email string `json:"email,omitempty"`
    pwd   string `json:"-"`          // never serialized
}

Comparing Structs

// Structs are comparable if all their fields are comparable.
a := Person{"A", 1}
b := Person{"A", 1}
fmt.Println(a == b)   // true

Empty Struct

Methods

A method is a function with a special receiver argument. It binds the function to a type.

Basic Method

type Rectangle struct { W, H float64 }

// receiver appears between func and the name
func (r Rectangle) Area() float64 {
    return r.W * r.H
}

r := Rectangle{3, 4}
fmt.Println(r.Area())

Value Receiver vs Pointer Receiver

type Counter struct { n int }

// VALUE receiver — operates on a COPY
func (c Counter) Get() int { return c.n }

// POINTER receiver — modifies the original
func (c *Counter) Inc() { c.n++ }

c := Counter{}
c.Inc(); c.Inc(); c.Inc()
fmt.Println(c.Get())   // 3

When to Use Each

  • Use pointer receiver if the method modifies the receiver, or if the type is large (avoid copy), or to keep the method set consistent.
  • Use value receiver for small, immutable-feeling types (like time.Time).
  • Be consistent within one type — mixing receivers is confusing.

Methods on Non-Struct Types

type MyInt int

func (m MyInt) Double() MyInt { return m * 2 }

var x MyInt = 5
fmt.Println(x.Double())   // 10
You can only define methods on types declared in the same package. You cannot add methods to int directly — wrap it in your own type first.

Interfaces

An interface is a set of method signatures. Any type that implements those methods satisfies the interface — there is no implements keyword. This is "structural typing", and it's the heart of Go's polymorphism.

Defining and Satisfying

type Shape interface {
    Area() float64
    Perimeter() float64
}

type Circle struct { R float64 }

func (c Circle) Area() float64      { return 3.14 * c.R * c.R }
func (c Circle) Perimeter() float64 { return 2 * 3.14 * c.R }

// Circle implicitly satisfies Shape — no declaration needed
var s Shape = Circle{R: 5}
fmt.Println(s.Area())

The Empty Interface

Any type satisfies interface{}. Since Go 1.18, any is an alias for interface{}.

var i any = 42
i = "hello"
i = []int{1,2,3}

Type Assertions

var i any = "hello"

s := i.(string)              // panics if i is not a string
s, ok := i.(string)          // safe — ok is false if not
if !ok { fmt.Println("not string") }

Type Switch

func describe(i any) {
    switch v := i.(type) {
    case int:
        fmt.Printf("int: %d\n", v)
    case string:
        fmt.Printf("string of len %d\n", len(v))
    case nil:
        fmt.Println("nil")
    default:
        fmt.Printf("unknown type %T\n", v)
    }
}

Interface Composition

type Reader interface { Read(p []byte) (int, error) }
type Writer interface { Write(p []byte) (int, error) }

type ReadWriter interface {
    Reader
    Writer
}

The Stringer Interface

type Stringer interface {
    String() string
}

type User struct { Name string; Age int }

    return fmt.Sprintf("%s (%d)", u.Name, u.Age)

Error Handling

Go has no exceptions. Errors are values returned from functions, just like anything else. The convention: the last return value is an error.

Panic, Defer, Recover

For truly exceptional, programmer-error cases, Go has panic. For cleanup, defer. To intercept a panic, recover. Use these sparingly — errors should still be the default.

defer

Schedules a call to run when the surrounding function returns. Defers run in LIFO order.

func demo() {
    defer fmt.Println("1 — runs last")
    defer fmt.Println("2")
    defer fmt.Println("3 — runs first")
    fmt.Println("body")
}
// Output:
// body
// 3 — runs first
// 2
// 1 — runs last
Arguments to a deferred call are evaluated at the defer statement, not when the deferred function actually runs.
i := 10
defer fmt.Println(i)   // captures 10 right now
i = 20
// Prints 10, not 20

panic

func mustOpen(name string) *os.File {
    f, err := os.Open(name)
    if err != nil {
        panic(err)
    }
    return f
}

A panic stops normal execution: deferred functions still run, then the program crashes with a stack trace.

recover

Stops a panic. Only useful inside a deferred function.

func safeDivide(a, b int) (result int, err error) {
    defer func() {
        if r := recover(); r != nil {
            err = fmt.Errorf("recovered: %v", r)
        }
    }()
    return a / b, nil     // b=0 panics with runtime error
}
Don't use panic/recover for ordinary control flow. Use it for unrecoverable issues (programming bugs, missing required configuration), or to convert a panic into an error at API boundaries.

Goroutines

A goroutine is a lightweight thread managed by the Go runtime. Starting one costs only a few KB of stack. The go keyword runs any function call as a new goroutine.

Starting a Goroutine

func say(msg string) {
    for i := 0; i < 3; i++ {
        fmt.Println(msg, i)
    }
}

func main() {
    go say("hello")   // runs concurrently
    say("world")
}
When main returns, all goroutines are terminated. You must coordinate with channels or sync.WaitGroup.

Anonymous Goroutine

go func(name string) {
    fmt.Println("hi", name)
}("Ana")

Closure Capture Trap

// Pre-Go-1.22: classic loop variable bug
for i := 0; i < 3; i++ {
    go func() { fmt.Println(i) }()   // might print 3 3 3
}

// Fix: pass as parameter
for i := 0; i < 3; i++ {
    go func(i int) { fmt.Println(i) }(i)
}
Go 1.22 changed loop semantics so each iteration has its own i, fixing this for new code. The pattern above is still useful for clarity and older Go.

Channels

Channels are typed conduits for sending and receiving values between goroutines. The Go mantra: don't communicate by sharing memory; share memory by communicating.

Creating a Channel

ch := make(chan int)        // unbuffered
ch2 := make(chan int, 10)  // buffered, capacity 10

Send & Receive

ch <- 42          // send
v := <-ch          // receive
v, ok := <-ch      // ok = false if channel closed and empty
close(ch)          // done sending — only the sender should close

Unbuffered vs Buffered

  • Unbuffered — send blocks until another goroutine receives. Synchronizes the two.
  • Buffered — sends fill the buffer; block only when buffer is full. Receives block when buffer empty.

Example — Worker Returning a Result

func main() {
    ch := make(chan int)

    go func() {
        ch <- 2 + 3      // send result
    }()

    fmt.Println(<-ch)   // 5
}

Range Over a Channel

func producer(ch chan<- int) {
    for i := 0; i < 5; i++ { ch <- i }
    close(ch)
}

ch := make(chan int)
go producer(ch)
for v := range ch {
    fmt.Println(v)
}

Channel Direction

func send(ch chan<- int)   { ch <- 1 }   // send-only
func recv(ch <-chan int)   { <-ch }      // receive-only

select — Multiplex Channels

select {
case v := <-ch1:
    fmt.Println("from ch1:", v)
case ch2 <- 42:
    fmt.Println("sent to ch2")
case <-time.After(1 * time.Second):
    fmt.Println("timeout")
default:
    fmt.Println("no channel ready")
}

Common Patterns

// Done signal — close to broadcast
done := make(chan struct{})
go worker(done)
close(done)   // every worker receiving from `done` unblocks at once

// Fan-out / fan-in
results := make(chan int)
for _, job := range jobs {
    go func(j Job) { results <- process(j) }(job)
}
Sending to or closing a closed channel panics. Receiving from a closed channel returns the zero value immediately. Sending to a nil channel blocks forever.

Context

The context package carries deadlines, cancellation signals, and request-scoped values across API boundaries and goroutines. It is the standard way to propagate cancellation in Go.

Creating Contexts

import "context"

ctx := context.Background()           // root, never cancels
ctx = context.TODO()                 // placeholder during development

ctx, cancel := context.WithCancel(ctx)
defer cancel()

ctx, cancel = context.WithTimeout(ctx, 2 * time.Second)
defer cancel()

ctx, cancel = context.WithDeadline(ctx, time.Now().Add(5 * time.Second))
defer cancel()

ctx = context.WithValue(ctx, "userID", 42)

Listening for Cancellation

func work(ctx context.Context) error {
    for {
        select {
        case <-ctx.Done():
            return ctx.Err()    // context.Canceled or context.DeadlineExceeded
        case v := <-jobs:
            handle(v)
        }
    }
}
Context is conventionally the first parameter of any function that may block: func Foo(ctx context.Context, ...).

Generics

Generics (introduced in Go 1.18) let functions and types operate on any of a set of types — using type parameters and constraints.

Generic Function

// T is a type parameter; "any" is the constraint (no restriction)
func Map[T, U any](s []T, f func(T) U) []U {
    out := make([]U, len(s))
    for i, v := range s { out[i] = f(v) }
    return out
}

doubled := Map([]int{1,2,3}, func(x int) int { return x*2 })
words   := Map([]int{1,2,3}, strconv.Itoa)

Type Constraints

// Constraint = an interface listing the allowed types or required methods
type Number interface {
    ~int | ~int64 | ~float64     // ~T means "any type whose underlying type is T"
}

func Sum[T Number](xs []T) T {
    var total T
    for _, x := range xs { total += x }
    return total
}

Built-in Constraints

import "cmp"
import "slices"

// "comparable" — types that support == and !=
func Equal[T comparable](a, b T) bool { return a == b }

// cmp.Ordered — anything that supports < <= > >=
func Min[T cmp.Ordered](a, b T) T {
    if a < b { return a }
    return b
}

Generic Types

type Stack[T any] struct {
    data []T
}

func (s *Stack[T]) Push(v T)    { s.data = append(s.data, v) }
func (s *Stack[T]) Pop() (T, bool) {
    if len(s.data) == 0 { var z T; return z, false }
    n := len(s.data) - 1
    v := s.data[n]
    s.data = s.data[:n]
    return v, true
}

Best Practices

Idiomatic Go is consistent, simple, and explicit. A few cultural norms worth absorbing:

Style

  • Always run gofmt (most editors do it on save). There is no debate about formatting.
  • Use camelCase for unexported, PascalCase for exported.
  • Short receiver names: r *Rectangle, not this or self.
  • Acronyms keep their case: ID, URL, HTTP — write userID, not userId.

Errors

  • Handle errors at the call site — don't ignore with _ unless you really mean it.
  • Wrap context: fmt.Errorf("read config: %w", err).
  • Don't panic in libraries; return errors.
  • Compare errors with errors.Is / errors.As, not == on strings.

Concurrency

  • Don't start a goroutine you can't stop. Pass a context.Context.
  • Prefer channels for communication, mutexes for protecting state.
  • Run go test -race regularly.
  • Never touch a channel from outside the goroutine that owns it (the one that closes it).

API Design

  • Accept interfaces, return structs.
  • Make the zero value useful (bytes.Buffer, sync.Mutex work without init).
  • Keep interfaces small — io.Reader has one method.
  • Avoid setters/getters unless they actually do work.
